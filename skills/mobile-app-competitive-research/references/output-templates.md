# Output Templates

Use these templates when the user wants the full end-to-end deliverable set.

## 1. Competitor Report

Recommended filename:
- `reports/YYYY-MM-DD-<slug>-competitive-research.md`

Required sections:
- target summary
- data sources and windows
- key facts
- positioning
- feature breakdown
- hidden features / hidden behaviors from reviews
- user-loved features / why users like using it
- monetization model
- UA strategy
- main creative directions
- creative links
- ASO strategy
- user scale
- geography
- personas / demographics
- technical stack / engineering implications
- strengths
- weaknesses
- risks
- what a new entrant must do to beat it

## 2. Product Doc

Recommended filename:
- `researches/products/YYYY-MM-DD-<slug>-ai-product-doc.md`

Required sections:
- product thesis
- target user
- jobs-to-be-done
- pain points inherited from competitor category
- user-loved behaviors worth preserving
- AI feature strategy
- non-AI table stakes
- differentiation
- key user flows
- monetization
- growth loops
- roadmap
- success metrics

## 3. Tech Spec

Recommended filename:
- `researches/products/YYYY-MM-DD-<slug>-tech-spec.md`

Required sections:
- architecture overview
- client app modules
- backend services
- AI systems
- data model
- ingestion / playback / indexing pipelines when relevant
- privacy / security / compliance
- analytics / observability
- release phases
- technical risks
- build order
